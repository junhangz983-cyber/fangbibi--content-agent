# 段落化路由

方比比内容业务采用段落化推进。不要默认完整输出。

## 选题段

适用输入：
- 对标视频、标题、评论区、热点、灵感。
- 用户说“看这个能不能转成比比选题”。
- 用户只有方向，还没有真实采访素材。

输出：
- 素材内核判断。
- 可转成比比的选题方向。
- 优先级排序。
- 下一步需要比比确认什么真实经历。

不输出：
- 不直接生成完整采访问题。
- 不重构文案。
- 不生成最终标题。

## 采访提问段

适用输入：
- 已经选定 1 个选题。
- 需要给编导一组提问。
- 用户强调要打开比比表达欲。

输出：
- 选题复述。
- 前置提问。
- 细节追问。
- 情绪追问。
- 收束提问。
- 采访记录提示。

不输出：
- 不替比比写答案。
- 不根据假设经历生成成稿。
- 不提前定最终标题。

## 重构段

适用输入：
- 已有原始采访逐字稿。
- 用户说“重构一下”“剪一下”“整理成口播稿”。

输出：
- 核心爆点提取。
- 删除说明。
- 重构后的逐字稿。
- 可进入标题段的内容锚点。

不输出：
- 不改写原文。
- 不补连接句。
- 不把标题当最终标题。

## 标题段

适用输入：
- 已有成稿、重构稿或明确内容锚点。
- 用户说“起标题”“选标题”“标题优化”。

输出：
- 标题锚点判断。
- 标题候选表。
- 最推荐 3 个。
- 是否建议进入流量预测自检。

不输出：
- 不回头重构正文，除非用户明确要求。
- 不为弱稿硬凑爆款标题。

## 流量自检段

适用输入：
- 已有最终拍摄稿和最终标题。
- 团队准备发布前判断胜率。

输出：
- 是否建议进入 cheat-score。
- 是否建议进入 cheat-predict。
- 最该验证的假设。
- 预测时重点观察的维度。
- T+3d 复盘应看什么。
