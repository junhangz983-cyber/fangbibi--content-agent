# 逐字稿重构

## 最高规则

所有进入成稿的内容，必须逐字逐句来自原始逐字稿。

只允许：
- 删除无效内容。
- 调整原文片段顺序。
- 分段排版。
- 保留必要的编导提问。

禁止：
- 改写任何一句话。
- 替换任何一个词。
- 补充任何连接句。
- 合并两句话生成原文不存在的新句子。
- 用自己的话总结、升华、解释。
- 把意思相近的句子改成更顺的表达。
- 跨 take 合并拼接。

判断标准：如果一句话不能在原始逐字稿中一字不差地找到，就不能进入成稿。

用户确认最终版本、最终收束句或指定“最后是这个”时，以用户确认的原句为最高优先级；不得因为同素材里还有其他可用句、金句或升华句，就继续追加。

## 删除

删除：
- 拍摄现场废话。
- 设备、声音、网络、补拍讨论。
- 重复试错。
- 口误改口。
- 无效闲聊。
- 编导内部评价。
- 和主题无关的发散。

编导提问不是一律删除。能推动视频发展、提出关键问题或增强真实感的编导提问可以保留。

## 结构

按四段式组织：

1. 钩子：最能提出问题或制造兴趣的原句。
2. 矛盾 / 真相：具体事件、真实细节、冲突点。
3. 反转：认知变化、情绪转折、出乎意料的回答。
4. 升华：原文中已有的金句、价值观或情绪落点。

时间轴按 180 字 / 分钟估算。

## 输出格式

```markdown
## 一、核心爆点提取

每条必须引用原文原句，并说明为什么能打动妈妈群体。

## 二、删除说明

| 删除内容 | 删除原因 |
|---|---|

## 三、重构后的逐字稿（带时间轴）

**(0:00-0:10) 【钩子：XXX】**

**(0:10-0:25) 【矛盾/真相：XXX】**

**(0:25-0:40) 【反转：XXX】**

**(0:40-0:55) 【升华：XXX】**

## 四、可进入标题段的内容锚点

- 最强动作：
- 最强原话：
- 最强反差：
- 最强金句：

## 五、自检记录

- 原文约束：
- IP 一致性：
- 爆款潜质：
- 定稿回收：
```

## 补丁检查

使用方式：先按上面的重构规则完成初稿，再按本检查逐条修正。不符合补丁规则时，必须回到原文重做，不允许用解释糊过去。

### 一、剪辑式重构补丁

方比比逐字稿重构不是改写、润色、创作，而是“剪辑式重构”。

成稿正文只允许：
- 删除。
- 分段。
- 调序。
- 保留有效编导提问。

成稿正文禁止：
- 口语润滑。
- 规范化表达。
- 压缩合并。
- 补连接句。
- 补总结句。
- 补升华句。
- 为了成片感改词。
- 把两句话合成原文不存在的新句子。

判断标准：正文里任何一句，只要不能在原始逐字稿中一字不差地找到，就判定失败。

### 二、可用原句池补丁

输出成稿前，先在脑中建立“可用原句池”。

优先进入原句池：
- 能推动视频发展的编导提问。
- 能增强真实感的编导提问。
- 比比的完整回答。
- 情绪转折点。
- 具体生活细节。
- 原文已有金句。
- 能推动妈妈群体共鸣的原句。

如果用户已经明确指定最终句或最终段，只能围绕该指定版本做删减、分段、调序；不要从原句池里再补其他句子。

不要直接进入写作模式。

### 三、编导提问补丁

编导提问不能一律删除。

可以保留：
- 提出核心问题的编导提问。
- 推动回答继续展开的编导提问。
- 增强真实采访感的编导提问。

必须删除：
- “再说一遍”“看着我”“举个例子”等拍摄调度。
- “结尾 OK 了吧”“状态很好”等现场评价。
- 重复试探。
- 内部创作讨论。
- 无关追问。

### 四、现场废话补丁

以下内容必须删除：
- 设备、声音、网络、充电宝、WiFi、录音讨论。
- 补拍讨论。
- 状态评价。
- 情绪安抚。
- 现场调度。
- 纯背景铺垫。
- 情绪断裂点。
- 只为了凑时长的铺垫。

### 五、IP 红线补丁

必须检查：
- 不写婚恋话题，如“女朋友”“结婚”“找对象”。
- 不让爸爸抢戏。
- 父亲负面、父母冲突、父亲嫌贵、不做家务等必须删除或模糊。
- 不写爹味说教。
- 不把比比写成炫富、装成功、教育别人。
- 不把妈妈写成爱炫耀、爱索取、强势控制的人。
- 不泛泛夸妈妈伟大，必须落到具体生活细节。

father 出现规则：
- 低频、正向、让妈妈有生活代入感的提及可以保留。
- 非必要因果关系，用“家里”“父母”模糊。
- 父母具体相处日常、父亲负面行为、家庭冲突必须删除。

### 六、细节筛选补丁

只保留位于情感转折点上的细节。

保留：
- 能推动情感升级的具体画面。
- 能让妈妈粉代入的生活细节。
- 能形成反差或金句的原文。

删除：
- 单纯交代背景的叙述。
- 不推动主线的趣味闲聊。
- 只为了凑时长的铺垫。

### 六点五、多段故事取舍补丁

当同一条原始逐字稿里出现多段故事、多个案例或多个可成片方向时，不要默认全部保留。

必须先判断每段故事的张力：
- 是否有更强的冲突或反差。
- 是否有更具体的动作、物件、数字或原话。
- 是否更能让妈妈粉代入。
- 是否更符合方比比真诚、低调、孝顺的人设。
- 是否更容易形成钩子和闭环。

处理规则：
- 选择张力最强的一段故事作为主线。
- 其他故事只有在能服务主线、补足反转或增强真实感时，才少量保留。
- 不为了“信息完整”保留多个故事。
- 不把多个故事并列堆叠成流水账。
- 不跨故事拼出原文不存在的新因果。

判断标准：一条成片只需要一个最强故事主线。多个故事都好时，优先选择最能产生情绪转折和传播钩子的那一段。

### 七、时间轴补丁

时间轴按 180 字 / 分钟估算。

建议时长：1-1.5 分钟，约 180-270 字。

情感浓度和结构完整度优先，不为凑时长保留弱句。

### 八、输出前强制自检

输出前必须逐条检查：

- 成稿中是否有任何一句不是原文？
- 是否有任何词被替换、润色或顺口化？
- 是否有任何连接句、总结句、升华句是自己补出来的？
- 是否有把两句话合并成原文不存在的新句子？
- 是否跨 take 合并拼接？
- 是否误删了能推动视频发展或增强真实感的编导提问？
- 是否保留了现场废话、重复试错或拍摄讨论？
- 是否违反婚恋、father、爹味说教等 IP 红线？
- 如果用户确认了最后一句，是否严格停在用户确认的最后一句，没有再追加其他可用句？
- 如果原文有多段故事，是否已选择张力最强的一段做主线，而不是并列保留多个故事？

### 九、历史复盘补丁

以下是多轮比比素材重构后验证过的经验，后续必须作为 check 标准执行。

#### 做得好的方向

- 把重构从“写作模式”拉回“剪辑模式”：只剪原话，不写新话。
- 把“意思来自原文”升级为“逐字逐句完全一致”。
- 先建立可用原句池，再组四段式，不要直接生成成稿。
- 有效编导提问要保留，尤其是能提出核心问题、推动回答、增加真实采访感的问题。
- 四段式只是组织工具，不是创作许可；正文仍然必须全部来自原文。
- 好稿子的判断标准是：全是原文、主线清楚、真实感强。
- 家庭关系类素材优先收在具体动作闭环上，而不是抽象价值观上。比如“我也敢给他们打钱，他们也敢收了”比“有了更多对话的空间”更像最终成片。
- “火了”“年入百万”“打钱”等容易炫耀的素材，要把重心落到关系变化和父母接受度，不落到成绩展示。
- 多段故事素材要做取舍，不要并列保留。先选张力最强、最能闭环的一段做主线，其他故事只作为辅助，不做第二主线。

#### 曾经做得不好的地方

- 把“原文约束”执行成“原意约束”，导致句子看似合理但原文没有。
- 为了顺口，把病句、口语、重复表达改顺。
- 为了结构完整，补连接句、总结句、升华句。
- 把两句原文拼成一句原文不存在的新句子。
- 在没有逐句核对原文的情况下，形式化写“原文约束通过”。
- 为了四段式完整，牺牲原文真实。
- 早期误把所有编导提问都删掉，削弱了视频推进和真实感。
- 用户定稿删掉的抽象句、解释句、价值观句，不要在后续版本里自动补回去。
- 不要把“具体关系动作”改成“抽象关系解释”。例如不要把“我敢给，他们敢收”扩写成“我们有了更多对话空间”。
- 看到多个故事都可用时，不要因为舍不得而全部保留；多个故事并列会削弱主线张力。

#### 改进方法

每次重构必须按这个顺序执行：

1. 识别素材编号和核心主题。
2. 如果出现多段故事，先给每段故事判断张力，选出主线故事。
3. 围绕主线故事建立可用原句池。
4. 删除现场废话、设备讨论、重复试错、无效闲聊。
5. 保留有效编导提问。
6. 用原句池组织四段式。
7. 逐句核对原文。
8. 再输出固定格式。

### 十、定稿回收检查

每次用户给出最终定稿后，必须回看当前稿和定稿的差别，记录：
- 哪一处更短
- 哪一处更顺
- 哪一处更闭环
- 哪一处更像最终成片
- 哪一处更符合原文
- 哪一处是我自己多补的

这些结论不是备注，是后续重构的强制 check。

### 十点五、多轮定稿回收与审美对齐机制

触发条件：
- 同一个逐字稿或同一条素材，用户连续三轮以上表达不满意。
- 用户反馈包括但不限于：“不好”“不对”“还是虚构”“重来”“不够”“这个最后定了”“最后是这个”。
- 用户随后给出最终版、定稿版或明确确认某个版本。

触发后必须做两件事：

1. 分析用户最终版
   - 用户保留了哪些原句？
   - 用户删掉了哪些我曾经保留的句子？
   - 用户把哪一句放到了开头？
   - 用户把哪一句放到了结尾？
   - 用户保留了哪些口语重复、笑点、停顿或不完整表达？
   - 用户舍弃了哪些看似完整但张力弱的故事？
   - 用户是否选择了更具体的动作，而不是抽象价值观？
   - 用户是否缩短了铺垫，提前进入冲突或反差？

2. 沉淀审美 check
   - 写出“用户需要什么”：例如更短、更具体、更像成片、更有钩子、更保留口语、更少解释。
   - 写出“需要舍弃什么”：例如抽象总结、多个故事并列、过度升华、顺口改写、弱铺垫、重复解释。
   - 把结论加入后续同类素材的 check，不允许只口头总结。

输出格式：

```markdown
## 定稿回收与审美对齐

### 用户最终版保留了什么

### 用户最终版舍弃了什么

### 和我上一版的关键差别

### 后续必须写入 check 的规则
```

强制判断：
- 如果用户最终版比我上一版更短，后续优先学习“删了什么”，不要只学习“保留了什么”。
- 如果用户最终版保留了口语重复，不要自动润色掉同类重复。
- 如果用户最终版用具体动作闭环，不要再补抽象价值观解释。
- 如果用户最终版只选一段故事，不要再把多个故事并列保留。
- 如果用户最终版停在某个原句，不要因为还有其他金句就继续追加。

#### 定稿回收案例：火了以后给爸妈打钱

用户最终定稿保留：
- “我敢于给他们打钱了啊”
- “他们才愿意把这个钱给点了”
- “现在他们打开我的视频，什么都看到，什么都知道了”
- “我也敢给他们打钱，他们也敢收了”

用户最终定稿删掉：
- “得体的、合理的，他们能接受的理由”
- “现在他们点开我的账号就看得到了”
- “我跟他们有了更多对话的空间”

后续 check：
- 这类素材的核心不是解释“亲子关系变好了”，而是保留“给钱 / 点钱 / 敢给 / 敢收”的具体动作链。
- 结尾优先停在“双方都敢做某个动作”的闭环上，不要补抽象升华。
- 重复口语如果形成节奏可以保留，例如“今天发生什么好事情啦”重复一次；不要为了顺口自动删成书面表达。
- “火了”只能作为家庭接受度变化的背景，不要扩成成就展示。

#### 最终判定

只要出现以下任意一种情况，本版重构失败，必须重做：

- 原文没有的句子。
- 原文有但词被替换。
- 原文句子被顺口化、书面化、文案化。
- AI 补的连接句、总结句、升华句。
- 为了结构强行拼出来的段落。
- 有效编导提问被误删。
- 现场废话、重复试错或拍摄讨论被保留。
