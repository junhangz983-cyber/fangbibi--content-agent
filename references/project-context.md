# 方比比团队版内置项目参考

本文件把方比比项目中和“逐字稿重构”直接相关的项目资料压缩进 Skill 内部。团队安装本 Skill 后，不需要访问橘娃本机项目路径。

使用边界：
- 本文件只用于校准人设、受众、取舍优先级、爆款潜质和 IP 红线。
- 重构稿正文仍必须全部来自用户提供的原始逐字稿。
- 不能用本文件里的案例、金句或标题去补写原文没有的内容。

## 一、人设基准

方比比是 2001 年潮汕人，年入百万 00 后创业者，“比比先生”主理人。

核心人设：
- 有钱但不张扬。
- 孝顺懂事。
- 情商高，爱笑。
- 低调修心，厚积薄发。
- 会赚钱但不炫耀。
- 能把普通家庭小事讲出温度。

内容核心不是展示成功，而是通过年轻人视角讲亲子关系、妈妈、金钱观、家庭教育和代际理解。

### 家庭成员处理

妈妈是情感故事的绝对主角。

妈妈关键词：
- 小学文化。
- 情商高。
- 勤劳持家。
- 爱干净。
- 爱笑。
- 不凶孩子。
- 不主动索取。
- 嘴上说不要但心里在意。
- 低调体面，把好东西留给孩子。

爸爸规则：
- 爸爸可以低频、正向出现，用来增加真实家庭质感。
- 非必要时用“家里”“父母”模糊。
- 父亲负面、父母冲突、父亲嫌贵、不做家务、家庭对立必须删除或弱化。
- 爸爸不能抢走妈妈主线。

外婆可以作为独立亲情线处理，适合隔代亲、节俭、偏爱、见面越来越少等主题。

## 二、妈妈粉受众

核心受众是 45-65 岁女性妈妈粉，以 50-60 岁为核心。

她们不是来看“年轻人炫耀成功”，而是看：
- 这个孩子为什么这么懂事。
- 妈妈的付出有没有被看见。
- 自己作为妈妈是否值得被爱。
- 孩子长大后能不能理解妈妈。
- 能不能转发给自己的孩子看。

高共鸣来自：
- 妈妈被孩子看见。
- 孩子终于懂事。
- 妈妈的付出不是理所当然。
- 子女想起自己忽略妈妈的瞬间。
- 评论区能出现“我妈也是这样”“看哭了”“转给孩子看”。

高传播动机：
- 社交勋章：展示“教子有方”的价值观。
- 情感代偿：希望自己的付出也被孩子看见。
- 代际沟通：把视频转给孩子，当作自己说不出口的话。
- 焦虑对冲：通过优秀孩子的故事获得安慰和信心。

## 三、内容定位与语言

定位：

```text
两代人之间的桥梁
帮助妈妈理解年轻人，也帮助年轻人理解妈妈
```

情感基调：
- 真诚。
- 温暖。
- 治愈。
- 孝顺。
- 感恩。
- 不说教。

表达视角：
- 用第一人称“我”。
- 用孩子视角理解妈妈、心疼妈妈、想回报妈妈。
- 不用“你应该”，多用“我发现”“我觉得”“我后来才知道”。

口语特征：
- 保留真实语气词、停顿、重复。
- 重要口语重复如果形成节奏，可以保留。
- 不把病句、口误、真实口语自动润色成书面表达。
- 方比比语速按 180 字/分钟估算。

## 四、逐字稿重构硬规则

最高规则：

```text
成稿正文必须逐字逐句来自原始逐字稿。
```

只允许：
- 删除。
- 分段。
- 调序。
- 保留有效编导提问。

禁止：
- 改写。
- 同义替换。
- 口语润滑。
- 压缩合并。
- 补连接句。
- 补总结句。
- 补升华句。
- 跨 take 拼出原文不存在的新句子。

判断标准：
- 如果一句话不能在原始逐字稿中一字不差找到，就不能进入成稿。
- “意思来自原文”不等于“逐字逐句来自原文”。

## 五、四段式结构

常用结构：

```text
钩子 -> 矛盾/真相 -> 反转 -> 升华
```

钩子：
- 用原文里最能提出问题、制造反差或产生兴趣的句子。
- 编导提问如果能推动视频，可以保留。

矛盾/真相：
- 保留具体事件、真实对话、物件、数字、生活动作。
- 不保留单纯背景铺垫。

反转：
- 保留认知变化、情绪转折、出乎意料的回答。
- 不用自己补“所以其实……”这种解释。

升华：
- 只能使用原文中已有的金句、价值观或情绪落点。
- 如果原文没有升华，宁可短收，不补写。

## 六、删除清单

必须删除：
- 拍摄前闲聊。
- 设备、声音、网络、充电宝、录音讨论。
- 补拍讨论。
- 状态评价。
- 编导内部创作讨论。
- “再说一遍”“看着我”“举个例子”等拍摄调度。
- 重复试错。
- 无关发散。
- 情绪断裂点。
- 只为了凑时长的铺垫。

编导提问不是一律删除。

可以保留：
- 提出核心问题的提问。
- 推动回答继续展开的提问。
- 增强真实采访感的提问。

## 七、IP 红线

绝对禁止：
- 虚构经历。
- 婚恋话题，如女朋友、结婚、找对象。
- 爹味说教。
- 把比比写成炫富、装成功、教育别人。
- 把妈妈写成爱炫耀、爱索取、强势控制。
- 泛泛夸妈妈伟大，没有具体生活细节。
- 照搬其他博主爆款语录当成比比表达。

风险内容：
- 父亲抢戏。
- 家庭冲突变重。
- 把妈妈牺牲写得过苦，失去温暖感。
- 把直播/带货内容写成销售说明，失去亲情信任底层。

## 八、爆款母体与取舍参考

这些母体来自方比比近期验证过的爆款内容，只用于判断取舍和爆款潜质，不能用来补写正文。

### 1. 妈妈存在本身就是优点

代表：
- “最喜欢妈妈哪一点”
- 关键结构：提问 -> 限制条件 -> 超预期答案

取舍启发：
- 能让妈妈觉得“我这个人本身就值得被爱”的原句优先保留。
- 不能泛泛夸妈妈，必须有真实表达或反差。

### 2. 给妈妈打钱不是钱，是反向照顾

代表：
- “给父母打多少钱才够”
- “不敢给妈妈打钱”
- “到底给妈妈多大红包”

取舍启发：
- 金额、红包、打钱理由、妈妈收钱动作、妈妈后续反应优先保留。
- 只说“我要孝顺妈妈”传播力弱。
- “敢给 / 敢收 / 要理由 / 妈妈不点”这类具体动作链优先保留。

### 3. 赚钱后不敢告诉父母

代表：
- “赚了钱不敢告诉父母”
- “被妈妈揭穿赚太多钱”

取舍启发：
- “火了”“赚钱了”“年入百万”只能作为关系变化背景，不做炫耀主体。
- 高价值切面是家风、低调、不张扬、父母接受度变化。

### 4. 妈妈只教了一件事

代表：
- “母亲只教了一件事”
- “妈妈只做了一件事”

取舍启发：
- 从比比现在的品质，反推妈妈当年的具体动作。
- “夸孩子”“低调攒钱”“厚积薄发”“看见孩子的好”这类母教动作优先保留。
- 不要变成教育方法论讲课。

### 5. 潮汕女人 / 妈妈撑住家庭秩序

代表：
- “潮汕女人都很招财”

取舍启发：
- 地域标签、家务秩序、爱干净、勤快、会做人、爱笑、面相好等细节有辨识度。
- “妈妈每天做什么”比“妈妈是什么样的人”更重要。

### 6. 个人逆袭与家庭教育

代表：
- “被父母劝高三复读”
- “男生和女生谁更赚钱”

取舍启发：
- 老板、赚钱、创业类内容必须回到家风和妈妈教给他的底层能力。
- 只讲成功经历，妈妈粉代入会变弱。

### 7. 家庭关系真相

代表：
- “孩子不尊重妈的真相”

取舍启发：
- 可以借“关系真相”的结构，但少用父亲冲突，多回到妈妈动作和比比理解。
- 家庭痛点可以做钩子，但结尾必须回到温暖和理解。

## 九、过往情感口播案例取舍规律

过往 43 条情感口播里，能跑出来的内容通常有以下特征：

- 有具体节日、具体动作、具体物件：红包、牛肉丸、土豆丝、备忘录、成绩单、橘子、牛奶、茶具、碗筷。
- 有一个能让妈妈代入的日常委屈：做饭没人看见、等孩子回家、孩子不回消息、辛苦不说。
- 有孩子长大后的反向理解：以前不懂，现在知道妈妈为什么这样。
- 有母子身份反转：以前妈妈照顾我，现在我想照顾妈妈。
- 有轻松口语和真实笑点，但不能破坏情绪。
- 有“妈妈嘴上说不要，实际很在意”的行为反差。

弱内容通常是：
- 只有观点，没有物件。
- 只有孝顺态度，没有具体动作。
- 只是在夸妈妈，没有冲突和反转。
- 结尾过度拔高，像鸡汤。
- 情绪靠编导喂词，不像比比自己说出来的。

## 十、标题与内容锚点

本 Skill 不负责生成完整标题，但重构时要为标题段保留锚点。

标题锚点优先级：
1. 观众心里的问题，而不是作者的金句。
2. 具体动作，而不是抽象情绪。
3. 反常行为，而不是顺直觉孝顺。
4. 关系里的冲突，而不是完整叙述。
5. 能让妈妈想转发给孩子看的表达。

常见强锚点：
- 给妈妈打多少钱才够。
- 赚了钱不敢告诉父母。
- 不敢给妈妈打钱。
- 妈妈只做了一件事。
- 妈妈唯一的心愿。
- 妈妈的爱有多贵。
- 不打钱给妈妈了。

重构时要保留能支撑这些锚点的原文句子，但不能为了标题需要补写正文。

## 十一、五轮自检

第一轮：结构检查
- 钩子是否 8 秒内抓人？
- 矛盾/真相是否有具体细节？
- 反转是否有认知或情绪变化？
- 升华是否来自原文？

第二轮：原文约束
- 每一句是否逐字逐句来自原文？
- 是否替换、润色、顺口化？
- 是否补了连接句、总结句、升华句？
- 是否跨 take 拼接？

第三轮：IP 一致性
- 是否符合真诚、低调、孝顺、温暖？
- 是否让妈妈成为主角？
- father 出现是否必要？
- 是否有婚恋、爹味、炫耀？

第四轮：爆款潜质
- 是否命中过往爆款母体？
- 是否有具体动作、物件、数字、原话？
- 是否具备妈妈粉转发理由？
- 是否有新的角度或差异化切口？

第五轮：最终优化
- 能不能更短？
- 能不能更顺？
- 能不能更闭环？
- 是否停在最强原文落点，没有多补一句？

## 十二、团队执行提醒

团队版重构的目标不是“写一篇更漂亮的稿”，而是把原始逐字稿剪成一条更像方比比爆款的成片。

最终判断顺序：

```text
原文真实 > IP 一致 > 妈妈粉共鸣 > 爆款结构 > 表达顺口
```

只要“顺口”会破坏原文真实，就不要顺口。
