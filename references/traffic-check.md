# 发布前流量自检

方比比内容 Agent 负责把内容做对；`cheat-on-content` 负责把“我觉得会爆”变成可复盘的流量预测。

本层不是替代业务判断，而是在发布前增加外部校准。

## 接入流程

1. 内容 Agent 完成选题、采访、重构和标题后，产出最终拍摄稿。
2. 将最终稿放入方比比项目的 `scripts/` 目录。
3. 使用 `cheat-score` 对最终稿做轻量打分，判断是否值得进入正式预测。
4. 准备发布前，使用 `cheat-predict` 写盲预测日志。
5. 发布后记录链接和发布时间。
6. T+3d 回填播放、点赞、评论、转发等数据，做复盘。
7. 如果连续多次预测偏差同向，再升级方比比专属 rubric。

## 方比比专属流量自检维度

初始化方比比 rubric 时，不要照搬通用观点视频 7 维。建议使用：

| 维度 | 判断问题 |
|---|---|
| 妈妈粉代入 | 45-65 岁妈妈是否觉得“这说的是我” |
| 亲情投射强度 | 是否触发孩子懂事、妈妈被看见、养育回报 |
| 开头点击钩子 | 前 3 秒是否有反常、悬念或具体问题 |
| 情绪反差 | 是否有小时候不懂、长大才懂的反转 |
| 真实细节密度 | 是否有妈妈动作、原话、具体场景 |
| 标题开环 | 标题是否让人想问“为什么 / 然后呢” |
| 人设安全 | 是否低调、真诚、孝顺，无炫富、爹味、婚恋风险 |
| 转发理由 | 妈妈是否愿意转给子女或发朋友圈 |
