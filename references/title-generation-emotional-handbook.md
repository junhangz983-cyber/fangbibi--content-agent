# 情感口播标题手册

本文件内置 06 手册的可执行标题逻辑。生成方比比情感口播标题时以本文件为主，不依赖外部项目路径。

## 适用范围

用于妈妈、母子、亲情、红包、礼物、花钱、低调家风、潮汕妈妈、孩子长大、年入百万 00 后回报父母等情感口播内容。

默认标题锚点：

```text
年入百万00后 + 10字以内钩子
```

## 最高判断标准

标题只有两个结果指标：

- 会不会点击。
- 会不会转发。

标题不是概括文案，而是制造点击理由。它必须让观众产生“为什么”“然后呢”“他怎么做的”的缺口。

## 顶级原则

1. 不泄露结果
   - 标题只开环，不把答案讲完。
   - 写过程、反常、悬念，不写最终结论。

2. 双重浓缩
   - 信息浓度：人物关系、数字、物件、动作、场景要具体。
   - 情绪浓度：心疼、骄傲、被看见、愧疚、温暖要强。

3. 正向治愈
   - 可以有问题感和反差，但情绪底色必须温暖、爱、治愈。
   - 避免恐惧、焦虑、指责、审判、爹味说教。

4. 反直觉钩子
   - 打破“孝顺就是打钱”“赚钱就要让父母知道”“妈妈只关心钱”等顺直觉表达。
   - 优先使用反常行为承载情感：不敢打钱、不敢告诉父母、妈妈拒绝贵礼物、赚钱后更低调。

## 高点击结构

优先使用：

```text
人物关系 + 具体事件 + 反常冲突
```

常见入口：

- 妈妈 + 钱：打钱、红包、礼物、衣服、包、生活费。
- 妈妈 + 低调：不炫耀、不索取、怕孩子花钱、嘴上说不要。
- 妈妈 + 家风：只教一件事、一个习惯、一句提醒。
- 妈妈 + 孩子长大：终于懂事、开始回报、第一次看见妈妈。
- 妈妈 + 反常反应：发现孩子赚钱后的反应，和观众预期不一样。

## 标题模型

### 1. 认知倒置型

把常规孝顺认知反过来。

- 不写：我给妈妈打了很多钱。
- 写：不敢给妈妈打钱。

### 2. 情感归因型

表面讲钱或动作，实际讲妈妈的爱和家风。

- 钱不是重点，妈妈的反应才是重点。
- 礼物不是重点，妈妈为什么说不要才是重点。

### 3. 母爱成就感型

让妈妈粉感到“孩子长大后懂妈妈了”。

- 适合转发给孩子看。
- 适合激发“我妈也是这样”的评论。

### 4. 量化孝心型

用数字打开问题，但不要停在数字。

- 给父母打多少钱才够。
- 第一次给妈妈发红包后。
- 妈妈 300 块衣服都嫌贵。

### 5. 家庭博弈型

写家庭内部的温柔拉扯。

- 孩子想给，妈妈不要。
- 孩子想让妈妈享福，妈妈怕孩子辛苦。
- 孩子赚钱了，父母反而更低调。

### 6. 双向奔赴型

不要只写孩子孝顺，也要写妈妈如何保护孩子。

- 妈妈不主动要，但一直托举孩子。
- 孩子回报妈妈，妈妈先担心孩子。

## S 级标题方向

- 超预期回答型：问题很普通，答案出乎意料。
- 成长叙事型：从“不懂妈妈”到“终于懂了”。
- 价值观输出型：一句家风或妈妈的原则，但要有具体事件。
- 悬念揭秘型：先抛异常行为，再让正文解释原因。

## 避坑

- 不写空泛大词：伟大、感恩、温暖、幸福、治愈本身不是标题。
- 不把标题写成结论：妈妈真的很爱我、我终于懂妈妈了。
- 不把钱写成炫富：年入百万只是 IP 锚点，不是炫耀点。
- 不把妈妈写成索取型、控制型、爱攀比型。
- 不写婚恋，不让爸爸抢戏。

## 输出要求

每次输出标题时至少给：

- 5-10 个候选。
- 每个候选标注使用的模型。
- 最推荐 3 个，并说明点击理由。
- 如果素材弱，直接指出缺少哪个核心钩子。
