# 方比比逐字稿重构

## 使用边界

只处理方比比原始采访/口播逐字稿的重构。

做：
- 从原始逐字稿里抽取可剪片段。
- 删除现场废话、设备讨论、重复试错、无效闲聊。
- 保留能推动视频发展的编导提问。
- 按短视频成片逻辑分段、调序、加时间轴。
- 输出核心爆点、删除说明、重构稿、标题锚点和自检。
- 同一逐字稿在调用本 Skill 后，超过三轮仍被用户否定时，在用户给出最终版后自动做定稿回收，沉淀审美对齐 check，并写回本 Skill 的重构规则。

不做：
- 不做选题判断。
- 不设计采访问题。
- 不生成标题候选。
- 不做发布前流量预测。
- 不改写、润色、补连接句或补升华。

如果用户请求超出重构段，只完成逐字稿重构，并说明下一步应交给方比比总内容 Agent 或标题 Skill。

## 工作流

1. 先读 `references/project-context.md`，校准方比比人设、妈妈粉和内容边界。
2. 再读 `references/anti-patterns.md`，排除虚构、鸡汤、炫耀、爸爸抢戏等反模式。
3. 再读 `references/viral-mvp-baseline.md`，理解近期爆款母体和具体细节取舍。
4. 最后读 `references/transcript-restructure.md`，执行逐字稿重构。
5. 判断是否有足够原始逐字稿；没有原文时停止，不根据大意重构。
6. 建立可用原句池，只收原文中一字不差存在的句子。
7. 删除现场废话、设备讨论、重复试错、无效闲聊和内部评价。
8. 保留能提出核心问题、推动回答或增强真实感的编导提问。
9. 用原句池组织四段式：钩子、矛盾/真相、反转、升华。
10. 输出固定格式。
11. 逐句自检；如果任何正文句不是原文，必须重做。

如果同一逐字稿在调用本 Skill 后连续三轮以上被用户指出“不好”“不对”“还是虚构”“重来”“不够”“不准”等问题，后续用户给出最终版时，必须启动 `references/transcript-restructure.md` 中的“多轮定稿回收与审美对齐机制”：
- 分析用户最终版保留什么、舍弃什么、怎么开头、怎么收束。
- 判断哪些是用户真正需要的，哪些是后续必须主动舍弃的。
- 把结论写入本 Skill 的 check / 案例部分，让下一次同类素材自动按这个审美执行。
- 如果当前环境不能直接写文件，必须在回复里输出“待写入 Skill 的规则条目”，不能只口头复盘。

参考资料只用于判断取舍、结构和爆款潜质，不能用来补写正文。重构稿正文仍必须全部来自用户提供的原始逐字稿。

## 输出要求

默认输出：

```markdown
## 一、核心爆点提取

## 二、删除说明

## 三、重构后的逐字稿（带时间轴）

## 四、可进入标题段的内容锚点

## 五、自检记录
```

正文必须逐字逐句来自原文。分析、删除原因、标题锚点可以概括，但重构稿正文不能出现原文没有的字词。

## 校验脚本

如果用户提供了原始逐字稿文件和重构稿文件，优先运行：

```bash
python3 scripts/verify_verbatim.py --source 原始逐字稿.txt --draft 重构稿.md
```

脚本只做硬校验：检查重构稿正文行是否能在原始逐字稿中逐字找到。脚本不能替代内容判断。
